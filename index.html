How the Calculation Works

Data Sources

Total Tab
All 13,943 US orders in March 2026. Includes orders shipped from both US warehouses and China. Contains: destination state, SKU, quantity, weight, original warehouse, and carrier channel. Used as the full order universe for the estimation.
Partial Tab
9,009 orders that were actually shipped from US warehouses (LA/NJ) with real shipping cost breakdowns. Contains: dest zip, zone, base freight, fuel, residential, remote, and total cost per order. Used to calibrate surcharge multipliers and weight estimates.
Rate Card (US 2C Tab + Zone Tables)
Carrier pricing tables for COE 4PX, UNIUNI, Ontrac, Amazon, FedEx HD, and USPS GA — base freight by zone and weight. Zone tables (USLAXC / USNYCA) map zip codes to carrier zones per warehouse. Used to price every order under each scenario.
  1. Start with the total tab — all 13,943 March US orders. In reality, some shipped from China due to stock-outs, but we model the ideal scenario where all orders ship from US warehouses.
  2. Warehouse assignment: Each order is assigned to LA (USLAXC) or NJ (USNYCA) based on the destination state and the warehouse coverage map.
  3. Package weight: Estimated from the median weight of the same SKU in the partial tab (real US-shipped orders). If no SKU match exists, the overall median weight (2.9 kg) is used.
  4. Zone lookup: Using the USLAXC / USNYCA zone tables from the rate card. Since the total tab has state but no zip code, we simulate the carrier cascade at the zip-code level for every zip in the state, then take the weighted average cost. This accounts for partial carrier coverage — e.g. UNIUNI may only cover 60% of zips in a state, with the remaining 40% falling to the next carrier in the cascade.
  5. Current cost model: For US-shipped orders, we use the actual carrier from the channel field. For China-shipped orders, we apply the observed USPS GA leakage rate (26.4% of lightweight orders ≤1.5 kg) based on partial tab patterns. Orders assigned to USPS GA are priced at the USPS GA rate card. All others follow the cascade: COE → UNIUNI → Amazon → FedEx HD.
  6. Optimized cost model: Every order runs through the proposed cascade:
    COE covers postcode? → Yes: use COE / No → UNIUNI? → Yes / No → Ontrac? → Yes: use Ontrac / No → Amazon → FedEx HD → USPS GA (last resort)
  7. Surcharges: Amazon uses a calibrated multiplier (×) derived from real partial tab data (total Amazon spend ÷ modeled base freight). Ontrac uses the same multiplier as a conservative estimate (no Ontrac invoices yet). COE and UNIUNI have no surcharge. USPS GA has no surcharge.
  8. Saving = Current cost − Optimized cost, summed across all orders. Annualized = March × 12.

Key Assumptions & Limitations

  • No zip-code precision: state-level dominant zones introduce some error for states spanning multiple zones.
  • Ontrac surcharges are estimated: using the Amazon multiplier as a proxy. Once live invoices exist, this can be refined.
  • USPS GA leakage is modeled: for China-shipped orders, we apply the 26.4% leakage rate observed in US-shipped lightweight orders. Actual leakage may differ.
  • Annualization: assumes March volume is representative. Does not model seasonality or growth.
